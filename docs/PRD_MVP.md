# 電子寵物任務系統 PRD（索引頁）

> **本文檔已拆分為兩份獨立文檔。** 原先混合了 Web 產品需求與角色素材生產需求，現已正式拆開以避免混淆。

## 拆分後的文檔

| 文檔 | 涵蓋範圍 |
|---|---|
| [Product PRD](./product-prd.md) | 核心產品目標、使用者角色、頁面/模塊功能、任務規則、權限模型、商店道具、寵物系統（產品層）、部署策略、非功能需求 |
| [Asset / Character Pipeline Spec](./asset-pipeline-spec.md) | 角色設定、狀態資產需求、表情型/姿態型分類、素材命名與版本規則、一致性驗收標準、素材生產與微修流程 |

## 為什麼要拆

原始 PRD 把兩種不同層級的需求混在一起，導致：

- 產品功能已經能跑，但素材還不夠
- 素材工作流已建立，但產品展示方式未定

拆開後，產品開發與素材生產可以各自追蹤進度，互不阻塞。

## 原始 PRD 中的已知問題與處理方式

以下為原始 PRD 中自行標記的問題，在新文檔中的處理方式：

| 原始問題 | 處理方式 |
|---|---|
| 產品需求與素材需求混在一起 | 已拆分為兩份文檔 |
| 狀態需求未區分表情型/姿態型 | 已在 Asset Pipeline Spec §4 明確分類 |
| 道具需求不夠精確 | 保留在 Product PRD §9，未定義部分以 TODO 標記 |
| 角色一致性驗收標準未明文化 | 已在 Asset Pipeline Spec §6 正式定義優先順序 |
| 部署策略未正式落位 | 已在 Product PRD §13 正式寫入 |
